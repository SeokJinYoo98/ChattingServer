# AGENTS.md

## 작업 원칙

* 기본 작업 브랜치는 `codex/work`이다.
* `main` 브랜치에서는 코드 수정이나 커밋을 하지 않는다.
* 작업 전 현재 브랜치를 확인한다.

  * `git branch --show-current`
  * `git status --short`
* 사용자의 요청 범위 안에서만 작업한다.
* 요청을 넘어서는 기능 추가, 리팩토링, 파일 구조 변경, public API 변경, 설정 변경은 먼저 허락을 받는다.
* 요청과 무관한 파일, 공백, 줄바꿈, 포맷팅은 수정하지 않는다.
* `git push`, `git switch`, `git reset`, `git rebase`, `git commit --amend`는 사용자 허락 없이 실행하지 않는다.

## 검증 규칙

Codex는 사용자가 명시적으로 요청하지 않는 한 아래 명령을 자동 실행하지 않는다.

* `git diff`
* `git diff --check`
* `dotnet build`
* 솔루션 전체 빌드
* 공백 오류 검사

코드 수정 후에는 사용자가 확인할 검증 항목만 보고한다.

예시:

검증 필요:

* `dotnet build`
* `git diff --check`

빌드 실패나 공백 오류를 사용자가 알려준 경우, 해당 문제만 수정한다.

Git 커밋 규칙

코드 수정 작업이 완료되면 Git 커밋까지 진행한다.

관련 변경 파일만 git add 한다.
git commit까지만 수행한다.
git push는 수행하지 않는다.
코드 수정이 없는 작업은 커밋하지 않는다.
사용자가 “커밋하지 마”라고 요청한 경우 커밋하지 않는다.
main 브랜치에서는 커밋하지 않는다.

커밋 제목은 Conventional Commit 형식을 따른다.

형식:

type: 변경 내용 요약

타입:

feat: 기능 추가
fix: 버그 수정
refactor: 동작 변경 없는 구조 개선
chore: 설정, 기타 정리
docs: 문서 수정
test: 테스트 수정
style: 포맷팅
build: 빌드 설정, 패키지 변경
작업 완료 보고 규칙

코드 수정 내용에 대한 장문 보고는 작성하지 않는다.
변경 설명은 Git 커밋 메시지로 대체한다.

최종 응답은 아래 형식만 사용한다.

커밋 제목
실제 커밋 제목
커밋 내용
실제 커밋 본문
확인할 사항
사용자가 직접 확인해야 할 빌드, 실행, 공백 검사, 테스트 항목
미구현 항목이나 추가 확인이 필요한 내용
없으면 없음
